#### 求學經歷

**[專題研究 — 基於 RISC-V 架構的嵌入式安全模組開發]**
在大三期間，我參與了一項為期一年的專題研究，主題為「基於 RISC-V 架構的嵌入式安全模組開發」。該專案旨在為物聯網 (IoT) 裝置設計並實作一套輕量級、高安全性的硬體信任根 (Hardware Root of Trust, HRoT) 解決方案。我與指導教授及兩位同學合作，主要負責韌體層的設計與實作，以及部分硬體功能驗證。

我的主要工作包括：
1.  **安全啟動 (Secure Boot) 機制的韌體實作：** 使用 C 語言在 RISC-V 微控制器上開發了安全啟動韌體，確保只有經過簽章驗證的程式碼才能執行。我們採用橢圓曲線數位簽章演算法 (ECDSA) 進行驗證，並透過熔絲 (e-fuse) 儲存公鑰雜湊值，以防篡改。在測試中，我們成功在 Xilinx Artix-7 FPGA 開發板上搭建的 RISC-V 核心上實現了秒級啟動驗證，並有效阻擋了未經授權的韌體載入。
2.  **硬體亂數產生器 (Hardware Random Number Generator, HRNG) 的韌體介面與測試：** 與硬體組同學協作，設計了 HRNG 的韌體 API，並編寫測試程式驗證其隨機性與吞吐量。我們透過 NIST SP 800-90B 測試套件進行了驗證，確保所產生的亂數能達到加密安全等級，平均每秒產生 1MB 的高品質亂數。
3.  **安全儲存區 (Secure Storage) 韌體模組設計：** 為敏感資料（如加密金鑰、憑證）設計了基於加密檔案系統的安全儲存模組，利用 AES-256 GCM 模式進行資料加密，並透過硬體加速器進行加解密操作，平均讀寫延遲控制在毫秒級。
4.  **側通道攻擊防護策略評估：** 協助進行了基於功耗分析 (Power Analysis) 的側通道攻擊模擬，並初步評估了韌體層面（如亂數化延遲、常數時間操作）的防護效果。

透過這個專案，我深入理解了嵌入式系統的硬體與韌體協同設計、安全晶片設計的核心概念，以及在資源受限環境中實施加密與安全機制的挑戰。我學會了如何使用 Verilog 進行硬體介面描述、以 C/C++ 撰寫高效且安全的韌體，並掌握了故障分析 (Fault Injection) 與側通道攻擊 (Side-channel Attack) 的基本概念與防禦策略。

#### 動機
在「基於 RISC-V 架構的嵌入式安全模組開發」專題中，我親身參與了從硬體底層到韌體應用的整個安全設計流程。這個經驗讓我深刻體認到，儘管軟體安全日益受到重視，但硬體與韌體層的安全性才是整個系統安全鏈條中最關鍵，也往往是最容易被忽視的一環。當韌體或晶片本身存在漏洞時，上層的軟體防護將形同虛設。物聯網裝置的普及使得這些潛在的脆弱點數量呈爆炸式增長，從智慧家電到工業控制系統，韌體攻擊可能導致嚴重的經濟損失乃至生命威脅。

國立陽明交通大學資工所，在嵌入式系統、晶片系統與計算機結構、資訊與網路安全等領域擁有領先的研究團隊與豐富的學術資源。我特別關注貴所教授們在「安全晶片軟硬體系統設計」與「硬體特洛伊、安全與硬體特洛伊防護」等方向的研究，這與我的專題經驗和未來志向高度契合。我渴望能在貴所優異的學術氛圍中，進一步深耕韌體安全與安全晶片設計，學習更先進的防護技術與分析方法，例如形式化驗證 (Formal Verification) 於韌體安全中的應用，或是基於機器學習的硬體木馬檢測技術。我相信貴所的頂尖師資和實驗設備，將能幫助我將理論知識與實務能力提升至嶄新的層次，為建構更安全的數位世界貢獻心力。

#### 研究方向與動機
在物聯網時代，嵌入式裝置的安全性已成為一個刻不容緩的議題。由於這些裝置通常資源有限，且部署環境多變，使得傳統的軟體層安全防護難以全面覆蓋。我的專題經驗讓我深感，從硬體設計初期就融入安全性考量，並結合強健的韌體保護機制，是築建堅不可摧的防線的關鍵。

我的主要研究方向將聚焦於**「強化基於 RISC-V 的嵌入式系統韌體安全與新型態硬體安全防護」**。具體而言，我對以下子領域充滿熱情：

1.  **輕量級安全啟動與韌體更新機制：** 針對資源受限的 IoT 裝置，探索並設計更高效、更具韌性的安全啟動 (Secure Boot) 和遠端韌體更新 (Firmware Over-The-Air, FOTA) 協定。我希望能研究如何利用硬體安全模組，結合區塊鏈或分散式帳本技術，為韌體更新提供去中心化的信任驗證，從而降低中心化伺服器被攻陷的風險。這將涉及密碼學原語在嵌入式系統上的最佳化實作、韌體版本回溯攻擊 (Rollback Attack) 防護，以及空中介面安全性 (Over-the-Air Interface Security)。
2.  **側通道攻擊與故障注入攻擊防護：** 深入研究各種側通道攻擊（如功耗分析、電磁輻射分析）與故障注入攻擊 (Fault Injection Attack) 對於嵌入式系統韌體的威脅。我計畫探索在 RISC-V 處理器上，如何透過指令集擴充 (Instruction Set Extension) 或微架構設計，實現硬體層面的抗攻擊特性。同時，在韌體層面，研究如何引入更多亂數化、遮罩 (Masking) 與常數時間操作 (Constant-Time Operations) 等技術，以抵禦高階分析攻擊。
3.  **基於形式化方法的韌體安全驗證：** 傳統的測試方法難以完全覆蓋韌體的所有執行路徑，因此我對運用形式化驗證 (Formal Verification) 技術來證明韌體程式碼的安全性、無漏洞特性深感興趣。這包括使用模型檢查 (Model Checking)、符號執行 (Symbolic Execution) 等工具，對關鍵安全模組（如密碼模組、存取控制模組）進行嚴格驗證，從源頭上消除漏洞。

我深信，解決這些挑戰將大幅提升未來物聯網裝置的整體安全性。在貴所，我期望能結合現有的 RISC-V 開發經驗，利用貴所豐富的 FPGA/ASIC 設計工具鏈與實驗平台，實作並驗證我提出的安全架構與防護機制。我特別希望能有機會參與到如「安全晶片軟硬體協同設計」相關的研究專案，將我對韌體安全的熱情轉化為具體的技術貢獻。
