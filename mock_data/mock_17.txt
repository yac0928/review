### 求學經歷

#### 專題研究：基於 LoRaWAN 的智慧農業環境監測與預警系統
在大三升大四期間，我參與了系上的專題研究，開發一套旨在提升農業生產效率與作物健康的智慧環境監測系統。面對傳統農業數據採集效率低、人工成本高的痛點，我主要負責系統的通訊模組設計與後端數據處理。

我與團隊成員共同設計並實作了一套基於 LoRaWAN 協議的無線感測器網路。在硬體層面，我們選用 ESP32 微控制器，整合了土壤濕度、環境溫度、濕度及光照強度等感測器，並開發了低功耗的感測器節點韌體，確保在偏遠地區能長時間穩定運行。在通訊層，我負責優化 LoRaWAN 協議的排程機制，以減少封包碰撞並提升數據傳輸可靠性。我們在學校實驗農場部署了 15 個感測器節點，在實際測試中，系統在 2 公里半徑範圍內，數據傳輸成功率達到 97%，且感測器節點的電池續航能力可達 6 個月。

在後端部分，我負責設計數據匯流排，將透過 LoRaWAN 閘道器收集到的環境數據，利用 MQTT 協議發布至雲端平台。我進一步利用 Python 撰寫數據處理腳本，對原始數據進行清洗、校準，並儲存至時序資料庫 (如 InfluxDB)。此外，我還開發了基於機器學習的異常檢測模組，例如使用 Isolation Forest 演算法分析土壤濕度變化，當檢測到異常下降趨勢時，系統能自動發送簡訊通知農戶進行灌溉，有效降低人力巡檢頻率並預防作物缺水。此專案讓我深入理解了物聯網系統的端到端架構設計、低功耗通訊協議的實作細節，以及數據從採集到應用的完整流程。

#### 課程專案：分散式智慧家庭能源管理系統
在「分散式系統」課程中，我與兩位同學合作開發了一套分散式智慧家庭能源管理系統 (Smart Home Energy Management System, SHEMS)。該系統旨在優化家庭電力消耗，提升能源使用效率。

面對多樣的智慧家電與異質的通訊協議，我主要負責系統的架構設計與異質裝置的整合。我們採用微服務架構，將系統功能模塊化，例如數據採集服務、排程最佳化服務和用戶介面服務。為了整合不同廠牌的智慧家電（如支援 Zigbee、Wi-Fi 的裝置），我設計並實作了通用的轉接層 (Adapter Layer)，將不同協議的數據抽象化為統一的訊息格式，並透過 Apache Kafka 作為訊息佇列進行異步通訊。

能源管理的核心在於排程最佳化。我們利用 Python 實作了基於強化學習 (Reinforcement Learning) 的排程演算法，根據即時電價、用戶偏好及歷史用電模式，動態調整高耗能家電（如洗衣機、烘衣機）的啟動時程。透過與模擬器的協同工作，我們在模擬環境中證明，該系統能將尖峰時段的用電量降低 15%，並為用戶節省約 10% 的電費。這個專案讓我掌握了分散式系統的設計原則、訊息佇列的應用、異質系統整合的挑戰，以及機器學習在即時決策系統中的潛力。

#### 公民科技參與：g0v 社群「資料開放與視覺化」專案貢獻者
作為 g0v 社群的活躍成員，我貢獻於一個致力於提升地方政府數據開放透明度的專案。此專案的目標是將政府機關釋出的 CSV 格式原始數據，轉換為更易於大眾理解與互動的視覺化圖表。

我參與的具體工作包括：
1.  **數據清洗與標準化：** 針對多個縣市政府提供的不一致數據格式，我開發了自動化腳本（使用 Pandas 函式庫），對約 50 個不同的數據集進行清洗、轉換與標準化，將數據整合為統一的 JSON 格式。
2.  **API 開發與維護：** 協助建立並維護 RESTful API 介面，供前端開發者調用經過處理的數據，確保數據能即時、可靠地呈現。我利用 Node.js 與 Express 框架，實作了高併發處理能力的數據服務。
3.  **前端視覺化協作：** 與設計師及前端開發者合作，利用 D3.js 函式庫將數據轉換為互動式圖表，例如地理資訊圖 (Choropleth Map) 和時間序列圖，讓公民能直觀地了解城市發展指標。

透過參與 g0v，我不僅應用了程式開發、數據處理和 API 設計的技能，更學會了在非營利與志願性質的環境中，如何與跨領域夥伴（設計師、法律專家、公民記者）有效協作，以及如何將技術專長應用於提升公共利益。這段經歷深刻影響了我對資訊科技社會責任的理解。

## 研究方向與動機
基於大學期間在物聯網通訊與分散式系統的學習與實踐，以及對公民科技中數據可靠性的關注，我對於「**大規模物聯網低功耗廣域網路 (LPWAN) 的效能優化與安全性強化**」這一研究方向抱持著高度熱情與明確動機。

**研究問題動機：**
1.  **效能瓶頸：** 當前 LPWAN 技術（如 LoRaWAN, NB-IoT）在面對數以百萬計的裝置連接時，常遭遇網路碰撞、資源分配不均導致的數據延遲與丟包率上升等問題。尤其在我的智慧農業專案中，感測器節點增加到一定數量時，便開始觀察到數據傳輸的不穩定性。如何設計更高效的媒體存取控制 (MAC) 層協議，以實現大規模裝置的穩定接入與數據傳輸，是亟待解決的問題。
2.  **安全挑戰：** 資源受限的物聯網裝置難以部署傳統複雜的加密機制，使得 LPWAN 面臨諸多安全威脅，包括竊聽、數據篡改、裝置偽造等。我的公民科技經驗也提醒我，任何數據的傳輸都必須確保其完整性與真實性，尤其在公共基礎設施應用中更是如此。如何在有限的計算與能源預算下，為物聯網通訊提供輕量級且足夠強健的安全防護，是一個極具挑戰性的課題。

**擬議研究方向：**
1.  **基於機器學習的 LPWAN 資源管理與排程最佳化：** 探索利用深度強化學習 (DRL) 或聯邦學習 (Federated Learning) 技術，動態調整 LPWAN 網路中的頻率分配、傳輸功率和數據速率，以適應不斷變化的網路負載與環境條件。目標是開發一套能自主學習並優化網路資源利用率的智能排程機制，進而提升大規模物聯網的吞吐量與服務品質。我希望能夠將我在分散式系統中對排程最佳化的經驗，應用到無線網路資源管理上。
2.  **輕量級加密與異常行為偵測機制：** 針對 LPWAN 裝置的資源限制，研究適用於邊緣運算的輕量級密碼學方案（如橢圓曲線密碼學或基於雜湊函數的認證機制）。同時，開發基於裝置行為模式的異常偵測演算法（如基於序列模式挖掘或半監督式學習），即時識別惡意裝置的入侵或數據篡改行為，從通訊層面加固物聯網系統的安全性。我在智慧農業專案中嘗試過的異常檢測模型，可以作為這個方向的基礎。

我深信，貴所擁有堅實的網路通訊與資訊安全研究基礎，特別是針對物聯網應用領域的深入探討，將為我提供最前沿的知識和最完善的指導。我渴望在貴所老師的帶領下，運用我在專題與公民科技中所累積的物聯網系統實作經驗與數據處理能力，為建構一個更智能、更安全、更互聯的未來貢獻一份力量。
