### 求學經歷
**[專題研究 - 容器化應用程式安全分析框架之建構與效能評估]**
在大三下學期，我參與了指導教授的容器化應用程式安全分析專題研究。鑑於現代軟體開發日益採用微服務與容器技術，其安全性議題日益突顯。我的專案旨在建構一個自動化框架，用於檢測基於 Docker 與 Kubernetes 部署的應用程式中的潛在安全漏洞。

此框架整合了靜態應用程式安全測試 (SAST) 工具如 Semgrep 與 Bandit，進行程式碼層級的弱點掃描；同時也引入動態應用程式安全測試 (DAST) 工具如 OWASP ZAP，在運行時模擬攻擊，檢測常見的 Web 漏洞（例如 Cross-Site Scripting, SQL Injection）。我負責設計並實作一個中央協調模組，使用 Python 語言結合 Ansible 自動化腳本，實現對容器環境的自動部署、掃描工具的調用與掃描結果的整合分析。針對特定新型的 Common Vulnerabilities and Exposures (CVEs) 模式，我還開發了基於抽象語法樹 (AST) 分析的自定義規則。

透過為期八個月的專案實作，我們成功在一個包含五個微服務的模擬電商平台中，識別出超過 15 個高風險漏洞，包括未經授權的 API 存取與資料庫注入點。與傳統手動檢測相比，此框架將漏洞發現效率提升了約 40%，同時透過智慧型結果聚合，將誤報率降低了 25%，顯著提升了資安分析的效益與準確性。

**[實作專案 - 量子啟發式混合密碼系統之效率優化]**
在「密碼學與網路安全」課程中，我主導了一個小組專案，探討如何實現一個量子啟發式混合密碼系統。傳統的非對稱加密（如 RSA、ECC）在面對未來的量子計算機時存在潛在風險，而格基密碼學（Lattice-based Cryptography）被認為是具備量子抗性（Post-Quantum Cryptography, PQC）的潛力方案。

我們的專案目標是設計並實作一個混合密碼系統原型，其中金鑰交換部分採用 CRYSTALS-Kyber 演算法（一種格基密碼演算法），而數據加密則沿用傳統的對稱加密（如 AES-256 GCM）。我主要負責 Kyber 演算法的 C 語言實作與性能調優，包括使用 SIMD 指令集（如 AVX2）優化多項式乘法，並分析其在不同硬體平台上的加解密延遲與資源消耗。

最終，我們成功地在 Linux 環境下，使用 OpenSSL 庫與自行實作的 Kyber 模組，構建了一個安全的通訊通道。實驗結果顯示，在單次 Kyber 金鑰交換過程中，我們將其計算時間壓縮至約 1.2 毫秒，與參考實作相比，提升了近 15% 的效率，同時保持了與 AES-256 GCM 相當的數據傳輸速率。此專案讓我對量子抗性密碼學的數學基礎與工程實作挑戰有了深刻理解。

## 研究方向與動機
鑑於我在容器安全與量子抗性密碼學的實作經驗，我對「後量子密碼學的軟體實作安全與效能優化」以及「基於AI的智能安全攻防技術」這兩個研究方向抱有濃厚的興趣，並期望能在貴所深入探討。

**研究方向一：後量子密碼學 (PQC) 的軟體實作安全與效能優化**
隨著量子計算的發展，傳統非對稱密碼系統的安全基石受到嚴重挑戰。PQC 的標準化工作正加速進行，其中格基密碼 (Lattice-based Cryptography) 因其較好的效率和安全性被廣泛關注。我的專案經驗讓我體會到 PQC 演算法的複雜性及其對軟體實作性能與安全性的高度要求。我期望能深入研究：
*   **安全實作**: 探討 PQC 演算法（如 Kyber, Dilithium）在軟體層面的側通道攻擊 (Side-channel Attacks) 防禦機制，例如恆定時間執行 (Constant-time Implementation)、蒙哥馬利梯子算法 (Montgomery Ladder) 等技術，並評估其在現代處理器架構上的安全性。
*   **效能優化**: 針對 PQC 演算法中的核心運算（如多項式乘法、數論變換 NTT），利用處理器特有的指令集（如 ARM Neon, Intel AVX512）進行深度優化，以期在資源受限的嵌入式設備或高吞吐量的伺服器環境中，實現更快的加解密速度與更低的功耗。
*   **形式化驗證**: 探索將形式化方法 (Formal Methods) 應用於 PQC 演算法的實作驗證，以數學方式證明其安全性特性，避免引入潛在的實作漏洞。
我的動機源於對未來資訊安全威脅的深切關注，並期望能為全球 PQC 的部署與實用化貢獻一份力量。

**研究方向二：基於人工智慧的智能安全攻防技術**
隨著網路攻擊日益複雜與自動化，傳統基於規則的資安防禦體系面臨挑戰。人工智慧（特別是機器學習與深度學習）為資安領域帶來了新的機會。我的容器安全專案讓我意識到自動化分析的潛力，但我更希望進一步探討：
*   **AI 驅動的漏洞挖掘與利用**: 研究如何利用深度學習模型（如圖神經網路 GNN 或 Transformer）分析程式碼的抽象語法樹 (AST) 或控制流圖 (CFG)，自動識別潛在的零日漏洞模式，甚至自動生成概念驗證 (Proof-of-Concept, PoC) 的攻擊程式碼。這包括探索神經符號 (Neuro-symbolic) 方法，結合 AI 的泛化能力與符號執行的精確性。
*   **智能惡意軟體檢測與分析**: 開發基於深度學習的惡意軟體家族分類系統，通過分析惡意軟體的 API 調用序列、行為日誌或二進制代碼的嵌入表示，實現更精準、更具魯棒性的檢測，甚至能夠推斷其攻擊意圖。
*   **AI 自動攻防 (AI Red/Blue Teaming)**: 探索基於強化學習的智能代理，使其能夠自主學習攻擊策略（紅隊）或防禦策略（藍隊），並在模擬的網路環境中進行博弈，從而發現系統更深層的脆弱性並提升防禦韌性。
此方向的動機源於對提高資安分析自動化水平的渴望，我希望能夠開發出更智能的工具來應對不斷進化的網路威脅。

我深信，在交大資工所提供的豐沛學術資源與頂尖研究氛圍下，我將能夠結合我在軟體實作的經驗與對密碼學的熱情，深入鑽研這些前瞻性的資安議題。
