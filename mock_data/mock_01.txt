在大三期間，我擔任了教育部資安計畫的課程助教，協助指導超過 150 位來自全國各校的學員。我負責撰寫 C++ 與 Python 的資安實作教材，包含緩衝區溢位（Buffer Overflow）的原理示範與修補機制。這份經驗讓我學會如何將深奧的編譯器原理與記憶體配置知識，透過圖像化與具體的 Payload 攻擊範例清晰地傳達給他人。  [專題研究 - 產學合作：智慧化滲透測試框架開發與攻擊路徑分析]
大學專題期間，我參與了兩項與資安廠商合作的產學專案。首先是與科技公司合作開發「基於強化學習的網頁漏洞偵測系統」，我主要負責建構隔離測試環境（Sandboxing），並利用 Actor-Critic 演算法優化攻擊載荷（Payload）的生成效率，確保其能繞過動態偵測系統。
隨後，我轉向研究「大型語言模型於惡意程式代碼分析之應用」，探討 LLM 在自動化反組譯代碼分析中的準確性。我負責整理人工標註的惡意行為特徵，並研究如何過濾模型生成的幻覺性內容（Hallucination），提升靜態分析的自動化程度。在產學過程中，我體會到模型優化需從資料品質、架構調整與評估指標（如 False Positive Rate）三方面同步切入，且資料集的完整性對資安模型至關重要。  [跨領域學習]
除了資工領域，我積極修習法律學系關於科技法與智慧財產權的課程。透過跨域學習，我了解到數位攻防不只是技術競賽，有時也是法條解釋與合規性的碰撞。例如在修習「電腦犯罪偵查」時，我發現技術上可行的採證方式，在法律證據採信上卻有摺積般的複雜邏輯，而非單純的技術判定。這讓我學會從不同角度看待問題，尋求技術與法律的最佳解法。  動機
在參與產學專案的過程中，我發現如何從海量的網路流量與日誌數據中，精準提取出攻擊特徵是一個巨大的挑戰。在當今萬物聯網的時代，數據的消化與有效利用直接影響資安應變的速度。我也意識到自己在大型分散式架構下的資安理論尚有不足，而國立陽明交通大學數據科學與工程研究所的課程，結合了資訊學院的專業技術與跨領域應用，相信能幫助我提升處理異質、巨量數據的能力，以掌握更先進的智慧防禦方法。  研究方向與動機雖然 AI 輔助防禦已成為趨勢，但高品質的資安標註資料（如零日漏洞特徵）依然極為匱乏。當訓練資料不足時，自動化攻防模型難以在真實場景中產生理想結果。此外，資安資料涉及機密性，必須經過細緻的人工處理，但在面對海量攻擊樣本時，單純依賴人力顯得緩慢且容易出錯。  我曾嘗試利用 LLM 將數十筆原始攻擊指令擴展至上千筆生成樣本，但發現模型常出現語法錯誤或指令失效的問題。這讓我深信，訓練資料的品質研究與擴充技術對於模型效能具有關鍵性影響。我希望能深入探討如何利用有限的真實攻擊樣本，透過生成對抗網路（GAN）或提示詞優化技術，構建出高品質、具備多樣性的資安訓練資料庫。  讀書計畫短期（碩士入學前）持續追蹤 OWASP 最新資安趨勢，觀看進階黑客松實戰教學影片以了解攻防技術演進。  加強英文溝通能力，參與資安社群（如 HITCON）的志工活動，拓展技術視野。  修習 Linux Kernel 與網管基礎課程，為進階資安研究銜接準備。  中期（碩士在學期間）修習人工智慧與機器學習進階課程，並強化在「惡意程式自動化偵測」與「入侵防禦系統」上的實作能力。  專注於研究如何管理並從巨量流量中萃取出威脅情報，並將其應用於自動化攻防場景。  積極尋取大型資安公司實習機會，驗證理論於真實企業環境的可行性。  長期（畢業後）從事資安研究員或自動化攻防工程師相關工作。  持續追蹤前瞻加密技術與 AI 安全領域，為台灣資安技術防護貢獻所學。  其他加分文件摘要在校成績：歷年成績單顯示在資料結構、演算法、作業系統等核心學科皆為 A 或 A+。技術證明：具備 OSCP（Offensive Security Certified Professional）證照，熟練使用 Linux 與多種滲透測試工具。社群貢獻：SITCON 學生計算機年會編輯組員，負責技術文案校對與維護。
