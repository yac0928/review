# 自傳

## 引言：從程式碼到智慧環境的熱情

我是柯薇安，國立交通大學資訊工程學系的學生。從小對電腦科學抱持著一份無可救藥的熱情，這份熱情驅使我在大學期間積極投入軟體開發與系統建構，特別是對雲端計算與智慧環境的應用，更是產生了濃厚的興趣。我深信資訊工程不應只停留在實驗室的理論，更應透過實作解決真實世界的問題，建構能讓生活更便捷、更智慧的未來。這份信念也引導我一步步走向貴所，希望能在此深化專業知識，貢獻所學。

## 學術深耕：扎實的程式與系統基礎

在大學四年中，我戮力於學業，並從中奠定堅實的資工基礎。我的學業成績表現優異，整體修課平均達 **3.69**。在學期間，我深刻體認到扎實的基礎知識是攀登技術高峰的必要條件，因此無論是「資料結構」、「演算法」、「作業系統」或是「計算機網路」等核心課程，我皆投入大量心力學習，並積極參與課堂討論與實作練習。

-   **核心課程與實作能力：**
    -   在大二修習「物件導向程式設計」時，我以 **Java** 開發了一套簡易的圖形化檔案管理系統，熟練運用繼承、多型等概念，並透過多執行緒處理檔案操作，學會了軟體設計的模組化與高內聚、低耦合原則。
    -   在「作業系統」課程中，我與組員共同完成了一個基於 **Linux Kernel** 的行程排程器改進專案，旨在優化多核心環境下的 I/O 密集型任務效能，深入理解了行程管理、記憶體管理與同步機制。
    -   修習「計算機網路」時，我設計並實作了一個具備流量控制與錯誤恢復機制的應用層協定，使用 **Python** 模擬資料傳輸，讓我對網路協議的設計與實現有了更直觀的認識。

## 雲端航行：分散式系統與微服務實踐

為了將課堂所學付諸實踐，我從大三開始加入吳教授的「分散式系統與雲端應用實驗室」，深入研究雲端原生技術與分散式系統的應用。這段經歷讓我對雲端計算的挑戰與機會有了第一手的體驗。

-   **雲原生微服務架構專案 (111-112學年度)：**
    -   **專案主題：** 「**基於Kubernetes的智慧交通監控微服務平台建構**」。我們旨在開發一個可彈性擴展的系統，處理來自城市各處的即時交通數據。
    -   **職責與貢獻：** 我主要負責後端微服務的設計與開發，以及 **Kubernetes** 叢集的部署與維護。我獨立完成了路況分析與車流量預測的微服務，利用 **Python Flask** 框架與 **Redis** 進行快取管理。
    -   **技術棧：** 我熟練運用 **Docker** 進行服務容器化，使用 **Kubernetes** 實現自動部署、負載平衡與服務發現。為了確保系統的穩定性，我也導入了 **Prometheus** 與 **Grafana** 進行即時監控。
    -   **成果與反思：** 成功部署了可處理每秒高達 **5000 筆** 交通數據的測試原型，其平均延遲控制在 **100 毫秒** 以內。這次經驗讓我深刻體會到，在分散式環境中，服務間的通訊韌性與錯誤處理機制至關重要，同時也領悟到良好的 **CI/CD pipeline** (我們使用 **GitLab CI**) 對於加速開發與部署的巨大價值。

## 智慧環境探索：物聯網數據的邊緣處理

延續對雲端與分散式系統的興趣，我進一步將目光投向了智慧環境中的邊緣運算應用，期望能將雲端的彈性帶到數據源頭。

-   **智慧建築環境監控系統 (112-1 學期專題)：**
    -   **專案主題：** 「**基於Raspberry Pi與MQTT的智慧建築環境監控系統**」。目標是開發一套能即時監測室內溫濕度、空氣品質並能自動調節的系統。
    -   **職責與貢獻：** 我負責邊緣設備的程式開發與數據處理邏輯。我在 **Raspberry Pi** 上利用 **Python** 撰寫感測器數據採集程式，並透過 **MQTT** 協議將數據發佈至中央服務。同時，我也負責設計邊緣節點的數據初步篩選與異常檢測模組，減少上傳至雲端的資料量。
    -   **技術棧：** 除了 **Raspberry Pi** 與 **Python** 外，我也運用了 **SQLite** 在本地儲存短期數據，並使用 **Node-RED** 快速搭建了數據流與簡單的控制介面。
    -   **成果與反思：** 該系統成功在實驗室環境中穩定運行，實現了數據即時可視化與基於規則的自動控制。此專案讓我第一次體會到，在資源受限的邊緣環境中，如何高效地處理數據並兼顧系統的即時響應能力，是一個充滿挑戰但也極具潛力的領域。我意識到，將部分計算負載從雲端遷移至邊緣，能有效降低延遲，提升用戶體驗。

## 業界磨練：實習與敏捷開發

為了更全面地了解軟體開發的生命週期與業界實務，我把握機會在「拓星科技」擔任軟體開發實習生，參與實際的商業專案。

-   **拓星科技軟體開發實習生 (112年暑假，為期兩個月)：**
    -   **公司背景：** 拓星科技是一家專注於提供企業級雲端解決方案的新創公司。
    -   **職責與貢獻：** 我主要協助開發其內部雲端資源管理平台。我負責撰寫自動化測試腳本，使用 **Python Pytest** 針對後端 **RESTful API** 進行功能與效能測試。此外，我也參與了部分前端介面 (使用 **React** 框架) 的 bug 修正與功能迭代。
    -   **技術棧：** 我在這段期間接觸了 **Django REST framework**、**PostgreSQL** 以及 **AWS EC2/S3** 等技術。
    -   **成果與反思：** 我開發的測試套件協助團隊識別並修復了 **15 個** 潛在的 API 錯誤，並將測試覆蓋率提高了 **10%**。這次實習讓我親身體驗了敏捷開發 (Agile Development) 的流程，以及團隊協作、版本控制 (Git) 的重要性。我學會了如何在時間壓力下，以高效、高標準完成任務，並深刻理解了系統測試對於軟體品質的決定性影響。

    -   **雲端運算架構 (Cloud Computing Architectures)：** 探討雲端服務模型 (IaaS, PaaS, SaaS)、虛擬化技術、容器化與無伺服器 (Serverless) 架構的深層原理與優缺點。

-   **實驗室與研究準備：**
    -   **加入研究團隊：** 在第一學期結束前，我計畫積極聯繫並加入與我研究興趣相符的實驗室，例如吳育松教授的實驗室，深入參與其進行中的研究專案。
    -   **文獻回顧與工具熟悉：** 廣泛閱讀相關領域的頂級會議 (如 SOSP, OSDI, NSDI, EuroSys) 論文，理解目前研究前沿與未解決的問題。同時，熟練掌握更多雲端與邊緣運算工具，如 **Apache Kafka** 進行數據流處理、**Kubeflow** 進行機器學習工作負載管理、**Rust** 或 **Go** 進行高效能系統開發。

## 中期目標 (碩士班第二年：論文研究與成果發表)

第二年將是論文研究的核心階段，我將專注於選定的研究主題，力求取得創新成果，並目標在學術會議或期刊上發表研究成果。

-   **論文主題確立與研究：**
    -   **研究方向：** 我的論文預計將圍繞「**基於邊緣雲端協同的智慧環境即時數據處理與彈性服務部署**」展開。具體而言，可能探討：
        -   **異構邊緣節點資源管理與工作負載調度策略研究**：如何在不同運算能力的邊緣設備上，高效地部署與管理智慧環境服務，並與中心雲端協同。
        -   **智慧環境數據流的即時異常檢測與輕量級模型推論**：開發在資源受限的邊緣設備上，能進行高效且低延遲的感測器數據異常檢測與預測推論模型。
        -   **邊緣-雲端間彈性伸縮與服務遷移機制**：研究在智慧環境需求波動時，如何無縫地將服務在邊緣與雲端之間遷移，以優化資源利用與服務品質。
    -   **研究方法：** 將結合理論分析與系統實作，可能採用 **Kubernetes Edge (KubeEdge)** 或 **OpenYurt** 等邊緣雲端框架進行實驗驗證，並利用 **Python/TensorFlow Lite** 或 **PyTorch Mobile** 實現輕量級機器學習模型。
-   **系統原型開發與驗證：**
    -   設計並實作一套具備上述研究特點的智慧環境監控或管理系統原型，透過真實感測器數據或模擬數據進行效能評估與驗證。
    -   將使用多個邊緣設備 (如 Raspberry Pi 或 Jetson Nano) 搭建實驗平台，模擬真實智慧環境的異構性與分散性。
-   **論文撰寫與發表：**
    -   逐步撰寫碩士論文，與指導教授定期討論，修正研究方向與內容。
    -   目標是將研究成果投稿至 **IEEE Transactions on Cloud Computing (TCC)**、**IEEE Internet of Things Journal (IoT-J)** 等期刊，或 **IEEE ICDCS**、**ACM/IFIP Middleware** 等國際頂級會議。
