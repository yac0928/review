## 自傳

大家好，我是侯永誠，目前就讀於元智大學資訊工程學系。我一直以來對軟體工程領域抱持著高度的熱情，期望能透過在交通大學資訊工程學系研究所的深造，將我的能力推向更高的層次。

## 個人特質與求學經歷

回顧我的大學生涯，我認為自己是一個在團隊合作與人際互動中成長的學生。大學期間，我投入了相當多的心力在系上與校園活動的籌備。例如，我在大三時，曾參與「資訊倫理研討會」的籌備工作。這是一場為期半天的校級活動，參與人數達到約 60 人，主要負責聯繫講者、安排場地、製作宣傳海報，以及協調當天流程。過程中，我學會了如何有系統地規劃一個活動，並在多方意見中找到平衡點，確保活動能順利進行。雖然那段時間花在課外活動上的精力較多，導致我的學業成績（修課平均約 2.76）並不算特別突出，但我相信這些經歷讓我累積了寶貴的領導統御與溝通協調能力，也讓我更了解在真實情境中，如何與不同背景的人有效合作。

在學術方面，我在元智大學資訊工程學系主要學習了程式語言、資料結構、演算法、作業系統等基礎課程。我對軟體開發的流程和邏輯尤其感興趣，也很喜歡透過寫程式來解決問題。雖然技術深度還需要加強，但我一直努力理解程式背後的原理，並希望能將這些基礎知識應用到更複雜的系統開發中。我發現，光有程式碼的實作能力是不夠的，如何讓程式碼可維護、可擴展，並能符合專案需求，才是軟體工程的精髓。

## 專題與實作經驗

大學期間，我參與了一些課程專題，讓我能將理論知識付諸實踐。其中一個比較讓我印象深刻的是「大學社團管理系統」的開發。這個專題是與三位同學合作，我們嘗試使用 Python 的 Flask 框架搭配 SQLite 資料庫，來開發一個能讓學生查詢社團資訊、報名活動的網頁應用程式。

我在專案中主要負責前端頁面設計與使用者互動邏輯的實作。我們從需求分析開始，透過訪談學生與社團幹部，歸納出系統所需的功能，然後進行簡單的介面設計。在開發過程中，我們使用了 Git 進行版本控制，讓我初次體驗到團隊協作開發的挑戰與樂趣。例如，有時候會遇到程式碼衝突，或是邏輯不一致的問題，我們就必須花時間開會討論，找出解決方案。雖然這是一個比較基礎的專案，最終成果也還有很多可以改進的地方，但它讓我具體學習了從零開始搭建一個小型 Web 應用程式的流程，也體驗到軟體工程中「專案管理」與「團隊合作」的重要性。我認識到，再好的技術，如果沒有良好的溝通與協作，也很難產出符合預期的成果。

另一個經驗是參與「行動應用程式開發」課程專題，我們嘗試開發一個簡易的「待辦事項 App」。雖然我當時對 Kotlin 語言還不太熟悉，但我積極投入學習，透過閱讀官方文件與線上教學影片，摸索如何設計使用者介面、處理資料儲存。這個過程讓我了解到，面對不熟悉的技術，主動學習和不斷嘗試是克服困難的關鍵。

## 申請動機與未來展望

我之所以選擇交通大學資訊工程學系研究所，主要考量到貴所的學術聲望，以及在軟體工程與程式語言領域的卓越表現。交大資工所不僅擁有豐富的師資和研究資源，更是國內許多頂尖軟體人才的搖籃。我深知我的技術深度和研究經驗仍有許多不足，但我相信貴所能提供一個紮實的學習環境，讓我有機會補足這些不足，並拓展我的專業視野。

我特別希望能深入研究軟體工程領域中，如何優化開發流程、提升軟體品質與效率的相關主題。過去在課堂專案中，我常遇到團隊協作效率不佳、程式碼不易維護等問題，這讓我意識到僅僅會寫程式是不夠的，更需要具備良好的軟體工程方法與思維。我期待能學習到更進階的軟體架構設計、軟體測試、軟體專案管理等知識，並透過參與研究專案，實際應用這些概念。我認為，我在大學期間培養的良好溝通能力和組織協調經驗，能夠幫助我更好地融入研究團隊，並在專案執行中貢獻我的力量。

我渴望能在交大資工所這個頂尖的環境中，找到我真正感興趣且能持續投入的研究方向，並與優秀的師長和同學一同學習成長。

## 讀書計畫

### 短程目標（錄取後至入學第一學期）
-   **補強基礎知識：** 針對資料結構、演算法、作業系統等核心科目，透過閱讀經典教科書、參與線上課程（如 Coursera 或 Udacity）進行複習與加強，目標是提升對這些基礎概念的理解深度。
-   **熟悉研究環境：** 積極了解交大資工所的研究實驗室與其主要研究方向，特別是軟體工程、程式語言相關的實驗室。
-   **磨練程式能力：** 利用 LeetCode 或 HackerRank 等平台，持續練習程式解題，提升演算法實作能力與解題效率。
-   **英文能力提升：** 閱讀相關英文學術論文或技術文章，提升專業英文閱讀能力，為未來閱讀文獻做準備。

### 中程目標（第一學期後至畢業）
-   **修習專業課程：** 根據興趣與未來研究方向，選修高等軟體工程、軟體測試與品質保證、進階程式語言設計、機器學習導論等相關課程，逐步建構更廣泛與深入的專業知識。
-   **尋找指導教授：** 積極與感興趣的教授交流，了解其研究主題，並尋找適合自己的指導教授，確立碩士論文的研究方向，例如可能偏向於軟體專案管理方法、自動化測試工具的應用，或是某種特定程式語言在大型系統中的效能優化。
-   **參與研究專案：** 積極參與實驗室的研究專案或產學合作計畫，從實務中學習研究方法、問題解決技巧，並實際應用所學的軟體工程知識。
-   **撰寫碩士論文：** 在指導教授的指導下，逐步規劃、執行並完成碩士論文，預計會結合我的軟體工程與程式語言知識，針對一個具體問題提出解決方案並進行驗證。

### 長程目標（畢業後）
-   **投入軟體產業：** 希望能進入大型軟體公司或新創團隊，從事軟體開發、軟體架構設計或專案管理等職位，將在研究所習得的理論與實務經驗貢獻於產業。
-   **持續學習與成長：** 軟體技術日新月異，未來將保持對新技術的敏銳度，透過自學或參與研討會等方式，持續精進自身專業能力。
-   **回饋社會：** 期望未來能將自身所學回饋社會，透過參與開源專案、擔任講師或分享經驗，為資訊社群的發展盡一份心力。
