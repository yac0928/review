# 自傳

## 起點：對軟體開發與遊戲世界的熱情

我是李羿晨，來自元智大學資訊工程學系。從小便對電腦程式和遊戲的內部運作方式充滿好奇，這份好奇心驅使我踏上資工之路。大學四年期間，我努力學習基礎學科，修課平均維持在3.72左右，特別喜歡程式語言與軟體工程相關課程。我深信，好的軟體設計和高效能的程式碼，是創造卓越數位體驗的關鍵，尤其是在遊戲開發這個充滿挑戰與樂趣的領域。

## 技術成長：從程式語言到模組開發

在大學期間，我積極投入於各項技術能力的培養。除了紮實的課程學習，如資料結構、演算法、物件導向程式設計，我更花費許多時間在實作專案上。我熟練運用 C++、Python 和 Java 進行開發，特別是在遊戲相關專案中，我主要使用 C# 搭配 Unity 引擎。

大三下學期，我加入了「遊戲軟體設計實驗室」，這是我技術成長的重要階段。我在實驗室裡的主要任務是參與一個線上合作 RPG 遊戲的開發計畫。我負責其中一個核心的「AI NPC 行為模組」。這個專案要求我設計一套可擴展的行為樹系統，讓遊戲中的非玩家角色能根據情境做出不同反應。我與團隊成員利用 Git 進行版本控制，並遵循一些敏捷開發的原則，每週進行進度討論與功能疊代。

在開發過程中，我遇到了不少挑戰。最大的困難是確保 AI 行為邏輯的複雜性與遊戲效能之間的平衡。一開始，我設計的行為樹因為過於龐大，導致 NPC 在某些情況下會出現明顯的延遲。為了解決這個問題，我花了許多時間研究 Unity 的 Profiler 工具，並學習了如何優化 C# 程式碼，例如減少不必要的記憶體分配、利用物件池來重用頻繁生成的遊戲物件。這次經驗讓我深刻體會到，軟體架構的設計彈性以及效能最佳化在遊戲開發中的重要性，而不僅僅是功能實現而已。

## 展望未來：挑戰與精進

這些實戰經驗讓我對軟體工程和遊戲模組開發有了更深層次的理解。我學到了如何將抽象的設計理念轉化為實際可運作的程式碼，並在團隊協作中學習溝通與解決問題。我渴望能進入更具挑戰性的學術環境，深入探索軟體開發的尖端技術，特別是在複雜系統的設計與優化上。
