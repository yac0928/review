## 自傳

我是李哲維，來自國立中興大學資訊工程學系。從小，我便對解決問題充滿熱情，好奇事物運作的底層邏輯。大學期間，我逐漸從程式設計的基礎邏輯中找到樂趣，特別是對如何從龐雜資料中萃取有價洞見的領域充滿興趣，這也堅定了我投身巨量資料分析與資料探勘領域的決心。

## 我的學習與探索之路

我在大學四年累積的學業成績約為 3.24。大一初探程式設計，便對 Python 的靈活性印象深刻。大二時，透過「資料結構與演算法」的紮實訓練，以及「機率與統計」的數理基礎，為我日後深入資料科學打下根基。

大三是我在技術上快速成長的一年。我在「資料庫系統」課程中，不僅學習了 SQL 的語法與資料正規化，更在期末專題中，運用 MySQL 設計並實作了一個小型電商平台的資料庫架構，並透過 `explain` 指令優化了多個複雜查詢，將查詢時間從數秒縮短至毫秒級。

而真正讓我對資料科學著迷的，是「資料探勘」與「高維度資料分析」兩門課程。在「資料探勘」的專題中，我負責處理一個公開的信用卡盜刷資料集。我使用 Python 搭配 `pandas` 進行資料清理與特徵工程，嘗試了多種機器學習模型，如 `scikit-learn` 的 `Random Forest` 和 `XGBoost`，最終透過調整模型參數與 SMOTE 技術處理類別不平衡問題，將詐欺偵測的 F1-score 從 0.72 提升到 0.85。這個經驗讓我第一次體會到資料品質與特徵工程對模型效能的決定性影響，以及如何透過嚴謹的實驗設計來驗證假設。

在「高維度資料分析」課程中，我接觸到諸如 PCA、t-SNE 等降維技術。我在期末專題中，選用一個包含高維度基因表達數據的資料集，嘗試使用 Python 的 `scikit-learn` 和 `UMAP` 對其進行降維視覺化，並結合 `DBSCAN` 演算法進行無監督分群。我發現不同降維演算法對於保留資料結構與分群效果有顯著差異，這讓我對如何有效地從高維資料中提取本質資訊產生了濃厚興趣。

## 業界淬煉與實務洞察

為了將所學理論與業界實務結合，我於大三暑假，進入「創數科技」擔任資料工程實習生，為期三個月。這是我第一次近距離接觸真實世界的巨量資料應用。我的主要職責是協助資料管道的建置與維護，處理每日來自超過 10 個不同來源，約 50 GB 的用戶行為日誌資料。我主要使用 Python 撰寫 ETL 腳本，透過 `pandas` 處理非結構化日誌，並運用 SQL 語言優化資料庫的查詢效能，確保資料能準確且及時地流入數據倉儲，供後續分析使用。這次實習讓我深刻理解到，資料從採集、清洗到轉換的每一個環節都至關重要，並且第一次體會到系統測試與錯誤處理對於資料工程穩定性的重要性。透過與團隊成員的協作，我學會了在時間壓力下，如何排查問題並提出有效的解決方案。

## 從音樂到程式碼的多元學習

除了專業領域的探索，我同時也是吉他社的主唱。在社團中，我曾帶領樂團在多場校內外演出，包括 111 學年度校慶演唱會上，我們樂團成功吸引了超過 200 位觀眾。擔任主唱不僅培養了我的舞台魅力和表達能力，更讓我學會如何在團隊中溝通協調、面對壓力，並在最短時間內找出共同目標。音樂的結構和程式碼的邏輯在我看來有異曲同工之妙，兩者都講求層次、和諧與精準。這份跨領域的經驗讓我擁有更開闊的視野與解決問題的彈性思維。

總結大學四年，我從理論學習、專案實作到業界實習，一步步累積了在巨量資料與資料探勘領域的技術能力與實務經驗。我深知交大資工所擁有卓越的學術聲譽、頂尖的研究資源以及許多在資料科學領域領先的教授。我渴望能進入貴所，在更具挑戰性的環境中，與優秀的師長和同學們一同精進，探索高維度資料分析的深層奧秘，並將所學應用於解決真實世界的複雜問題。
