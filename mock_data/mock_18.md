## 創業與實戰經驗：從實作到反思

我的大學生活不只埋首書本，更積極投入實務專案與創業嘗試。大三上學期，我與幾位志同道合的同學共同創立了一個小型資料分析顧問工作室，專注於提供中小企業客製化的數據分析服務。
- **專案一：電商平台顧客行為預測**
  - **時間**：110-2 學期至 111-1 學期
  - **背景**：協助一家新創電商平台分析顧客的購買行為與流失風險，以優化行銷策略。
  - **執行細節**：
    - 負責建構資料管道，整合來自網站日誌、交易紀錄與社群互動等多源異質資料，總處理數據量超過 **1.5 TB**。
    - 使用 Python 中的 Pandas 進行資料清洗與特徵工程，從原始資料中提取如「平均停留時間」、「點擊率」與「購物車放棄率」等逾 **100 個**維度特徵。
    - 採用 scikit-learn 框架，實作了基於 Random Forest 與 XGBoost 的分類模型，預測顧客的流失機率。
    - 透過模型解釋性工具如 SHAP，分析重要特徵，協助客戶識別出影響顧客流失的關鍵因素，例如特定產品類別的低轉換率、長時間未登入等。
  - **成果**：我們的模型成功將客戶的顧客流失預測準確率提升至 **85%**，並協助客戶設計更精準的再行銷活動，使目標顧客的轉換率提高了 **12%**。
  - **反思**：儘管模型取得了不錯的成果，但我深感在處理高維度、具雜訊的真實資料時，僅憑經驗與試錯，難以確保模型在面對未見資料時的穩健性。我才意識到資料前處理與特徵選取背後的統計學意義，以及模型泛化能力的重要性，這些都需要更深厚的理論支持。

## 研究室與統計建模競賽：理論與實務的連結

除了創業經驗，我也積極參與學術研究與競賽，以補足實務經驗中欠缺的理論深度。
- **成大智慧決策實驗室研究助理**
  - **時間**：大三下學期至大四上學期 (111-2 學期至 112-1 學期)
  - **主題**：基於時間序列資料的異常行為檢測。
  - **執行細節**：
    - 在指導教授帶領下，我參與了針對智慧工廠生產線感測器資料的異常檢測研究。
    - 負責對每日生成約 **50 GB** 的多維度時間序列資料進行預處理，包括缺失值插補、異常值檢測與平滑化。
    - 學習並應用了如 **Isolation Forest** 與 **One-Class SVM** 等非監督式異常檢測演算法，並嘗試以 Autoencoder 搭配變分自編碼器 (VAE) 進行特徵提取與重建誤差分析。
    - 利用 Matplotlib 與 Seaborn 視覺化異常模式，協助工程師快速定位問題。
  - **成果**：在模擬環境中，我們的方法對特定異常模式的檢測召回率達到 **90%** 以上。
  - **反思**：這段研究經歷讓我首次體會到，嚴謹的學術方法對於解決複雜資料問題的關鍵性。我在處理高維時間序列資料時，常常需要回溯到論文中討論的統計假設與演算法原理，才能更精準地調整模型參數。這也讓我更渴望能系統化地學習相關的統計理論與先進的機器學習模型。

- **Kaggle 統計建模競賽**
  - **時間**：大四寒假 (112 年 1 月)
  - **主題**：房價預測競賽 (如 House Prices - Advanced Regression Techniques)。
  - **角色**：資料預處理與模型融合組員。
  - **執行細節**：
    - 負責對包含近 **80 個**特徵的數據集進行深度探索性資料分析 (EDA)，並處理遺漏值、異常值與類別特徵編碼。
    - 嘗試多種回歸模型，包括 Lasso Regression、Ridge Regression、ElasticNet 與 Gradient Boosting (LightGBM)，並利用 Cross-validation 進行模型評估。
    - 成功實作了堆疊集成 (Stacking Ensemble) 技術，將多個基學習器的預測結果進行二次學習，進一步提升模型效能。
  - **成果**：我們的團隊最終在來自全球 **3,000 多個**參賽隊伍中，獲得了前 **8%** 的成績。
  - **反思**：競賽的壓力與高度競爭，讓我體會到模型選擇、參數調優以及集成方法的重要性。但同時也暴露了我對這些方法背後數學原理與統計假設理解不足的問題，例如為何某些模型在高維資料上表現更優異，或如何更系統地選擇特徵以避免過擬合。
