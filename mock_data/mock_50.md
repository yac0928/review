**自傳**

我是廖昱晨，來自成功大學資訊工程學系。從我踏入資工領域的第一天起，對於如何運用程式碼創造價值、解決實際問題，就抱持著無比的熱情。回顧大學四年，我的學業成績大約是 3.15，或許這數字並非頂尖，但我深知這無法完全代表我對資訊領域的投入與付出。我大學生涯的重心，除了課堂學習外，更多地放在了實際的專案開發與嘗試創業上，透過這些跌跌撞撞的經驗，我才真正體會到理論基礎的重要性。

大二那年，我與幾位志同道合的朋友創立了一個小型團隊，最初是想嘗試接案，後來則將重心放在開發一個針對小型商家客戶行為分析的數據平台。我們意識到，許多在地小店雖然累積了大量顧客資料，卻苦於缺乏有效工具來解讀這些數據。於是，我們開始著手開發一個基於 Python 和 Flask 的輕量級系統，用來蒐集和視覺化銷售數據，並嘗試提供一些基本的顧客分群與推薦功能。在實作過程中，我主要負責後端資料處理與模型建構，運用了像是 Pandas 進行數據清洗、Scikit-learn 庫實現 K-Means 聚類和簡單的協同過濾推薦演算法。當時，雖然憑藉著一股衝勁和網路上的教學資源，我們成功地打造出了 MVP (最小可行產品)，也找到了一些願意試用的商家，但很快地，我們就遇到了許多瓶頸。

首先是效能問題。當數據量逐漸增加時，我們的系統處理速度明顯變慢，原本單機運行的架構根本無法負荷。我嘗試透過調整程式碼、優化資料庫查詢來解決，但治標不治本，最終發現這需要更深層的分散式系統和資料庫設計知識。再來就是模型表現的穩定性。雖然我們的推薦系統在小規模數據上看似有效，但在面對複雜多變的實際商業場景時，其準確性和泛化能力卻不盡理想。當時的我，對於背後的統計學理論、機器學習演算法的數學推導、以及高維度資料處理的優化策略，都只是一知半解，只能憑感覺調整參數，結果往往事倍功半，甚至出現過度擬合的問題。這段經歷讓我深刻體會到，雖然我累積了廣泛的技術應用經驗，從資料爬取、處理、建模到部署都曾涉獵，但在學術深度上仍有顯著的不足。

面對這些挫折，我沒有選擇放棄，反而更加堅定了要補足理論基礎的決心。我開始花更多時間在大學課程中鑽研，例如資料結構與演算法、演算法設計、機器學習導論等，努力去理解那些在實作中遇到的問題背後的數學與計算原理。我也參與了一些課程專題，例如在大三的「巨量資料分析」課程中，我與組員利用 Spark 框架處理一個大規模的公開文本數據集，並嘗試實作主題模型（LDA），雖然過程中對於參數設定和結果解讀仍感吃力，但至少讓我對分散式計算有了初步的認識。此外，我也自主學習了深度學習的基礎，透過 Coursera 等線上平台補充了卷積神經網路 (CNN) 和循環神經網路 (RNN) 的知識，並嘗試應用在一些影像分類的個人 side project 上，例如開發一個簡單的貓狗圖片分類器，用 TensorFlow 實作。這些經驗讓我對模型建構與設計的複雜性有了更清晰的認識，也更清楚地知道自己欠缺的是什麼。

創業的經歷讓我培養了解決問題的能力、快速學習新技術的適應力，以及將技術應用於商業場景的敏銳度。然而，我也清晰地認識到，這些「快」和「廣」的學習模式，無法取代扎實的理論基礎和系統性的學術訓練。我渴望能進入貴所，在更嚴謹的學術環境下，將我的技術知識進行更深層的系統化，補足過去在分散式系統、進階機器學習演算法、高維度資料分析等領域的理論不足。我深信，有了這些更深厚的學術涵養，我才能真正將我的技術能力提升到一個新的層次，未來無論是在學術研究或是產業應用上，都能做出更有價值的貢獻。

**讀書計畫**

我對交通大學資訊工程學系的嚮往，是基於其在人工智慧、巨量資料與分散式系統領域的卓越研究能量。我過去的創業與專案開發經驗，讓我看到了資料科技在商業應用上的巨大潛力，同時也點出了我在理論與系統化知識上的不足。因此，我申請貴所，核心目標是希望能夠系統性地深化我在巨量資料、高維度資料分析以及模型建構與設計方面的專業知識，並學習如何將這些知識應用於更複雜、更具挑戰性的問題。

在接下來的研究生階段，我會將學習重心聚焦於幾個主要方向。首先，我會積極修習如「高等機器學習」、「深度學習」、「統計推論」等核心課程，期望能從底層數學原理與演算法設計的角度，徹底理解各種模型的優缺點與適用情境。過去，我常憑直覺選擇模型，現在我希望能更科學地評估、選擇並優化它們。特別是對於處理高維度資料的技巧，例如維度縮減、特徵工程等，我希望能有更深入的學習，這對於我過往處理商業數據時，經常遇到特徵過多、資料稀疏等問題會有很大幫助。我也很期待能接觸到貴所教授們在圖神經網路（GNN）或強化學習（Reinforcement Learning）等前沿領域的研究，這些都令我感到非常興奮。

除了機器學習與資料分析本身，我也深知分散式系統與高效能運算在巨量資料處理中的關鍵地位。我計畫修讀「分散式系統」、「平行處理」等課程，希望能理解如何設計、實作與管理大規模的資料處理架構。我的創業經歷讓我對單機系統的瓶頸有切身體會，因此對於如何運用 Spark、Hadoop 等框架進行高效能的資料處理與計算，我抱有強烈的學習動機。這將直接彌補我在過去專案中因系統擴展性不足而遭遇的困境。我希望不只是學會使用工具，更能理解其背後的原理，甚至有能力進行底層的優化。

在選擇指導教授方面，我會積極與貴所對巨量資料、機器學習、高維度資料分析領域有研究的教授們聯繫，希望能深入了解他們的研究方向。目前我對黃〇〇教授在機器學習演算法最佳化、林〇〇教授在資料探勘與模式識別，以及張〇〇教授在分散式系統與雲端計算方面的研究非常感興趣，我相信這些研究領域都與我過去的經驗和未來想補足的知識缺口高度契合。我希望能有機會參與到這些教授的研究專案中，從中學習如何進行嚴謹的學術研究，包括問題定義、方法設計、實驗驗證與結果分析等環節。

至於論文研究方面，我希望結合我在商業場景的實戰經驗，探討如何將先進的機器學習模型，應用於解決實際產業中的巨量資料挑戰。舉例來說，針對高維度且具有時間序列特性的客戶行為數據，如何設計一個兼具預測準確性與可解釋性的推薦系統；或者在非結構化數據（如文本、影像）中，如何高效地提取有價值的資訊並進行模式識別。我期望能找到一個能兼顧理論深度與實務應用價值的研究題目，透過論文的撰寫過程，完整訓練我的學術研究能力。

我深信，在交大資工所這個頂尖的學術殿堂，我能夠獲得我所渴望的系統化知識和理論深度。我會以最嚴謹的態度投入學習與研究，期望兩年後，我能成為一個不僅能應用技術，更能理解、設計與優化複雜資訊系統的專業人才，為台灣乃至全球的資訊科技發展貢獻一份心力。
