### 求學經歷
**[專題研究：高可用分散式鍵值儲存系統的設計與實作]**
在大三升大四的暑假，我參與了系上與某新創公司合作的產學專題，旨在設計並實作一個具備高可用性與強一致性的分散式鍵值儲存系統。該系統需支援每秒處理超過 50,000 次讀寫請求，並確保在單一節點失效時，系統仍能正常運作且資料不丟失。我的主要職責是負責核心的共識演算法（Consensus Algorithm）與資料複製機制（Replication Mechanism）的實作。

我選擇以 C++ 語言在 Linux 環境下進行開發，並採用 Raft 共識演算法來維護資料的一致性。為達成高可用性，我們設計了三副本（3-replica）的資料複製策略，並透過心跳機制（Heartbeat Mechanism）與領導者選舉（Leader Election）來偵測與恢復失效節點。在實作過程中，我深入研究了非同步網路通訊模型（Asynchronous Network I/O），利用 `epoll` 等系統呼叫來處理高併發的網路事件，並優化了序列化與反序列化（Serialization/Deserialization）的效率，以減少網路傳輸的延遲。透過 `gRPC` 框架，我們實現了不同服務節點間的高效通訊。

為驗證系統效能，我設計了一系列壓力測試，使用 `YCSB` (Yahoo! Cloud Serving Benchmark) 模擬不同工作負載。最終，我們成功地在模擬的五節點叢集中，達成了平均讀寫延遲在 5 毫秒以內、單節點故障自動恢復時間小於 10 秒的目標，並展示了在單一節點失效時，系統吞吐量僅下降約 25% 的優異表現。這個專案讓我對分散式系統的複雜性、一致性模型、容錯機制以及性能調優有了深刻的理解，並熟練運用了 C++ 在系統級開發中的各種技巧。

**[課程專案：基於容器化技術的即時作業系統排程器優化]**
在「高等作業系統」課程中，我與團隊成員共同完成了一個課程專案，目標是優化基於 Linux 容器（Docker）的即時作業系統排程器，以滿足物聯網邊緣裝置對低延遲與穩定性的需求。我們聚焦於 `CFS (Completely Fair Scheduler)` 在混合工作負載（Mix Workloads，包含 CPU 密集型與 I/O 密集型任務）下的表現。

我負責設計並實作一個 C 語言的 Linux 核心模組，引入了一種基於任務權重與歷史資源使用率動態調整的排程策略。我們透過修改 `cgroup` 機制，為不同的容器設定彈性的 CPU 配額與優先級，並監控其在 `per-cpu` 上的運行時間與上下文切換次數。相較於標準的 `CFS`，我們的排程器在處理 10 個容器同時運行、其中 3 個為即時（Real-time）任務（例如影像辨識推論）的場景下，成功將即時任務的平均延遲降低了 15%，最大延遲降低了 20%，同時對非即時任務的影響在可接受範圍內（吞吐量下降約 5%）。此專案讓我對作業系統核心機制、排程演算法、容器化技術及其在邊緣計算中的應用有了實務經驗。

## 研究方向與動機
我期望在交大資工所深耕的研究方向主要聚焦於「**面向雲邊協同（Cloud-Edge Collaboration）的智慧型分散式資源管理與即時排程**」。

在當前物聯網（IoT）裝置爆炸性增長與人工智慧普及化的趨勢下，傳統的雲端集中式處理模式已逐漸無法滿足所有應用對低延遲和高頻寬的要求。邊緣運算（Edge Computing）因此應運而生，它將計算能力推向資料源頭，顯著降低延遲。然而，邊緣裝置的資源受限性、網路環境的異質性與不穩定性，以及資料在雲端與邊緣之間流動的複雜性，都對現有的作業系統與分散式系統提出了全新的挑戰。

我的研究動機主要來自於以下觀察：
1.  **異構與資源受限環境下的排程挑戰**：邊緣節點通常包含不同類型、算力不一的硬體（如低功耗 ARM 處理器、FPGA 加速器），且記憶體與儲存容量有限。如何有效地在這些異構且資源受限的裝置上，智慧地排程容器化應用程式（如 Docker, k3s）中的任務，並確保關鍵應用（例如即時影像分析、工業控制）的低延遲與穩定性，是一個亟待解決的問題。我曾透過 Linux 核心模組優化 CFS，對此領域有了初步認識，並渴望進一步探索基於強化學習或多目標最佳化的智慧排程演算法，以動態適應不斷變化的邊緣環境。
2.  **雲邊協同的資料一致性與容錯機制**：在雲邊協同架構中，資料需要在雲端與多個邊緣節點之間同步與聚合。這不僅涉及到如何設計高效且低耗能的分散式共識協議來保證資料一致性，還需考慮邊緣網路不穩定時的容錯與斷線操作（Offline Operation）能力。我的分散式鍵值儲存系統專案讓我理解了 Raft 演算法的原理，我希望進一步探索針對邊緣環境優化的輕量級共識協議，或基於 CRDTs (Conflict-free Replicated Data Types) 的最終一致性模型，以應對高延遲、低頻寬的邊緣通訊場景。
3.  **安全性與隱私保護**：邊緣裝置通常部署在不可信的環境中，且處理敏感資料。如何整合硬體安全模組（如 TPM），設計安全的作業系統架構與分散式存取控制機制，並在系統層面實現同態加密或安全多方計算的基礎設施支援，以保護邊緣資料的隱私，也是我極感興趣的跨領域議題。

我期待透過在交大資工所的學習與研究，能深入探討這些問題，開發出新一代智慧型的分散式作業系統與資源管理框架，為未來高可靠、低延遲的雲邊協同應用奠定堅實基礎。
