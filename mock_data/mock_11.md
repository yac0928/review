親愛的交通大學資訊工程學系研究所招生委員會：

我是王欣儀，目前就讀於國立臺灣大學電機工程學系。從小便對科技世界充滿好奇，特別是那些能實際解決問題、影響生活的技術。大學四年在電機系所受的訓練，讓我打下了紮實的基礎，從電路學的細節到微積分的抽象推導，都讓我對工程學有了深刻的理解。然而，隨著學習的深入，我逐漸發現自己對於軟體系統的架構與其背後邏輯更感興趣，尤其是那些能夠處理大規模資料、支援全球服務的分散式系統與雲端運算技術，這份好奇心驅使我開始跨足資訊工程領域。

其實，最初選擇電機系，是希望能夠從硬體的角度深入理解科技的本質。大一時，我認真修習了電路學、電子學與電磁學等基礎課程，對於物理世界的運作規則有了較為宏觀的認識。當時的學業成績表現穩定，修課平均一直維持在班上前列，最終達到 4.0 的水準，這也證明了我對於學習的熱忱與投入。然而，在一次偶然的機會下，我參與了一個關於智慧城市應用的小型專題，其中涉及到了大量感測器資料的收集、處理與儲存。當時我們嘗試使用現有的雲端服務平台來部署簡單的資料庫與應用程式，雖然只是初步的嘗試，但那個經驗讓我深深著迷於軟體所能帶來的無限可能性與彈性。我意識到，即使沒有深入硬體底層，純粹的軟體設計也能創造出巨大的價值，尤其是當它與網路結合，能夠服務成千上萬，甚至是數百萬的使用者時。那個專題成了我跨領域學習的一個重要轉折點，讓我開始主動探索資工領域的課程與知識。

從大二下學期開始，我積極選修了許多資訊工程學系的課程，像是「資料結構與演算法」、「作業系統」、「計算機網路」與「資料庫系統」等。這些課程讓我對電腦科學的核心概念有了系統性的理解。在「資料結構與演算法」的課程中，我學會了如何分析演算法的時間複雜度與空間複雜度，並在實際專題中應用了圖論演算法來解決網路路由問題，這讓我意識到演算法設計的精妙與其對系統效能的關鍵影響。而「作業系統」則讓我對處理器排程、記憶體管理與檔案系統有了基礎的認識，尤其是多執行緒與同步機制的挑戰讓我印象深刻。當時我們在課程專題中實作了一個簡單的 shell，雖然功能有限，但從頭建構一個小型系統的過程，讓我對軟體的層次架構有了更深的體會。

到了大三，我進一步選修了「分散式系統概論」與「雲端運算」等進階課程。在「分散式系統概論」中，我接觸到了容錯機制、共識演算法（例如 Paxos 的基本概念）以及分散式交易的挑戰，這些知識讓我對大型系統的穩定性與可靠性有了新的理解。當時的課程專題是設計一個簡化的分散式鍵值儲存系統，我與組員們使用 Python 搭配 gRPC 框架，模擬了資料的分片與複製，雖然只是概念性的實作，但從中學習到了如何處理網路延遲、節點故障等實際問題，這段經歷讓我對實際的分散式系統架構產生了濃厚的興趣。除此之外，在「雲端運算」課程中，我們學習了雲端服務模型（IaaS, PaaS, SaaS）以及主要的雲端供應商（AWS, GCP, Azure）提供的服務，並透過小組專題，將一個簡單的網路應用程式部署到 AWS EC2 實例上，並利用 S3 儲存靜態檔案。這些實務操作讓我更具體地理解了雲端環境的便利性與其背後複雜的基礎設施。

除了課堂學習，我也很注重實務經驗的累積。在大三暑假，我有幸在一家新創公司「藍星科技」擔任軟體開發實習生。這是一間專注於企業級雲端解決方案的公司。我在後端開發團隊中工作，主要負責維護與開發基於 Spring Boot 框架的 RESTful API 服務。在實習期間，我熟悉了 PostgreSQL 資料庫的操作，學會了如何撰寫高效的 SQL 查詢語句，並參與了將部分遺留系統遷移至微服務架構的專案。當時我們團隊採用敏捷開發模式，讓我體會到快速迭代與團隊協作的重要性。我還學會了使用 Git 進行版本控制，並透過 Docker 將服務打包成容器，部署到 Kubernetes 測試環境中。雖然只是一些輔助性的工作，但這些經驗讓我對業界的開發流程與常用技術棧有了切身的認識，也更加堅定了我未來在軟體工程領域深耕的決心。這段經歷讓我發現，將課堂所學應用到真實世界的問題上，不僅考驗技術能力，更考驗解決問題的思路與團隊合作的精神，確實獲益良多。

同時，我也積極參與校內的「開源社群」，擔任編輯組長一職。作為編輯組長，我的主要職責是統籌社群內技術文章的撰寫與校對，確保文章的品質與內容符合開源精神。我負責組織成員分享最新的開源技術資訊，將複雜的技術概念轉化為易於理解的文字，並發布在社群部落格上。這份工作讓我培養了良好的溝通與協調能力，也讓我對開源文化有了更深的認同。我深信開源是推動科技進步的重要力量，而分散式系統與雲端運算領域的許多核心技術，如 Linux Kernel、Kubernetes、Apache Kafka 等，都源自於開源社群的貢獻。參與開源社群的經歷，讓我對這些技術的協作模式與發展邏輯有了更深層次的理解，也讓我更加期待能為這個領域做出自己的貢獻。

綜合電機系的數理邏輯訓練以及資工系的軟體開發實務，我認為自己具備了從不同角度分析問題的能力。電機系的背景讓我對系統底層的物理限制和效能考量有較好的直覺，而資工領域的學習則賦予我構建複雜軟體系統的工具與思維。我相信這種跨域的學習歷程，能讓我在未來投入分散式系統與雲端計算的研究時，擁有更獨特、更全面的視角。我的目標是成為一位能夠設計高效、可靠且具備良好擴展性的雲端系統架構師，為新世代的應用提供堅實的底層支援。

在讀書計畫方面，我期許自己能在貴所的優質環境下，進一步深化在分散式系統與雲端計算領域的專業知識，並將電機系所學的嚴謹思維與資訊工程的實務技能完美結合。我之所以選擇交通大學資訊工程學系，正是因為貴所在分散式系統、雲端運算與大規模資料處理方面有著卓越的學術聲譽與豐富的研究資源。我對於貴所許多教授在高效能計算、容錯系統及雲端平台優化等方面的研究深感興趣，期望能有機會跟隨具備相關專業的老師進行深入探討。

進入研究所的第一年，我將專注於修習核心課程，包括「高等作業系統」、「高等計算機網路」、「分散式演算法」以及「雲端服務架構」等。這些課程將有助於我建立更為紮實的理論基礎，補足在跨域學習過程中可能存在的知識縫隙。我也計畫深入學習 Go 語言，因為它在現代分散式系統開發中扮演著越來越重要的角色，其併發模型與高效能特性非常吸引我。同時，我會利用課餘時間鑽研更多關於 Kubernetes 與 Docker 的進階知識，包括其內部運作機制、叢集管理與服務網格（Service Mesh）等，並嘗試在個人專案中實作一些複雜的部署情境，例如利用 Prometheus 與 Grafana 監控容器化應用程式的效能，或是在測試環境中模擬服務的故障轉移。我相信這些實作經驗對於理解和優化實際的雲端系統至關重要。我也會積極參與實驗室的專題討論，與學長姐和教授交流，以便更快地融入研究環境，並探索自己未來碩士論文的研究方向。
